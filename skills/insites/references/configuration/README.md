# Configuration

## .insites File (Environment Endpoints)

Located at project root. JSON file generated by `insites-cli env add`. Defines environment credentials:

```json
{
  "dev": {
    "instance_uuid": "uuid",
    "token": "token",
    "email": "dev@example.com",
    "url": "https://your-instance.staging.oregon.platform-os.com",
    "key": "key"
  }
}
```

## app/config.yml (Feature Flags)

Application-level configuration:

```yaml
# app/config.yml
modules:
  core:
    enabled: true
  user:
    enabled: true
  common-styling:
    enabled: true
```

## Constants (Runtime Configuration)

For secrets and environment-specific values. See `references/constants/`.

```bash
insites-cli constants set --name KEY --value "value" dev
```

## Module Configuration

Modules are configured via installation and constants:

```bash
# Pull module from instance
insites-cli modules pull payments dev

# Configure module via constants
insites-cli constants set --name stripe_sk_key --value "sk_..." dev
```

> **CLI STATUS:** `insites-cli modules install` is not yet available. Module installation is currently done manually.
